Создание мини-игры на нативном javascript

Этот проект я выполнил в рамках изучения js. В нем реализовал: методы работы с DOM деревом, обработкой событий, а также наиболее часто используемые методы и свойства объектов.

  • Нативный JavaScript
  • Реализация мини-игры на js с возможностью самостоятельно установить время.
Более подробно прочитать, а так же сыграть ;) можно >> «Click me»
Исходные данные
  • Возможно задавать время (по умолчанию выставленно 5 секунд).
  • За указанный промежуток времени необходимо кликнуть по максимальному количеству генерируемых квадратов.
  • В процессе игры необходимо заблокировать input, для того что не было возможность сбить запущенный таймер.
  • Генерируемые div должны быть разного размера, цвета и появляться в заданной области.

Выпадающий список на JS

  • Нативный JavaScript
  • Реализация выпадающего списка на js, с возможностью гибкой кастомизации.

Создание выпадающего списка на нативном javascript

В данном проекте я реализовал выпадающий список на js, который похож на разметку html: section > option. Плюсом данного подхода является полная кастомизация и возможость адаптации под различные задачи.

  • Нативный JavaScript
  • Реализация выпадающего списка на js, с возможностью гибкой кастомизации.
Подробно прочитать, а также посмотреть на реализацию можно >> Dropdown on js
Исходные данные
  • Работа с данными. Список городов, в данном случае, храниться в массиве (для примера взял города миллионики), но возможно подгружать данные динамически.
  • Данные из массивов, при формировании элементов списка (<li>...</li>) забираются по data-id="...". Это оказалось очень удобно, при дальнейшем выводе блока со ссылкой на выбранный город.
  • Скрытие всплывающего списка реализовал различными способами: по нажатию клавиши "Escape", клику вне области списка и повторному клику по тегу с классом dropdown__label (в него передается выбранный город).

Сам макет вы можете скачать >> psd layout

Я сделал дополнительно две страницы:

  1. Страница блога – является каталогом с ссылками на статьи.
  2. Страница статьи – непосредственно шаблон для написания и публикации материала.
